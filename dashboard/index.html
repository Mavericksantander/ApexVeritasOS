AI Agent Governance MVP

The governance layer between agents and the world.

AVOS gives autonomous agents a real identity, auditable logs, a safety gate for risky actions, and a reputation signal humans can trust. Built for developer adoption: local-first, simple API, and an SDK.

Explore the API View dashboard External onboarding

What you get

  • Identity UUID + JWT
  • Safety gate /authorize_action
  • Reputation +0.5 / -1.0
  • Heartbeat active agents

Platform

Identity, authorization, and observability for autonomous agents.

Agent identity

Agents register once, receive a stable UUID, and operate under a session token. Every action can be traced back to the agent that requested it.

Endpoint POST /register_agent

Action firewall

Before executing risky actions, agents ask AVOS for a decision. Denied and verification-required actions are logged for later review.

Endpoint POST /authorize_action

Reputation signal

Track outcomes over time. Successful tasks increase reputation, failures reduce it, and humans can use the score as a quick trust heuristic.

API POST /log_task

Operational heartbeat

Agents report their status and version. AVOS surfaces which agents are active and correlates activity with tasks and authorization decisions.

Endpoint POST /agents/{id}/heartbeat

Flow

The end-to-end loop for agents and humans.

The AVOS loop

Identity → heartbeat → tasks → authorization → reputation → visibility

Agent side
  1. Onboard once. Get agent_id and a session token.
  2. Send heartbeat. Report model, version, status.
  3. Log tasks. Success: +0.5 reputation, failure: -1.0.
  4. Before risky actions, ask AVOS. /authorize_action returns allow/deny/verify.
  5. Execute only if allowed. Denied actions are blocked and recorded.
Human side
  1. Discover agents. Use public directory + search by capability.
  2. Observe activity. See who is active via heartbeat.
  3. Audit decisions. Review blocked actions and reasons.
  4. Trust signal. Reputation + task history inform delegation.
  5. Iterate policies. Tighten or expand guardrails as you learn.
Example: onboarding + authorization localhost / ngrok 
curl -s http://127.0.0.1:8000/external/register_agent \\\n  -H 'Content-Type: application/json' \\\n  -d '{\"developer_id\":\"demo\",\"bot_name\":\"openclaw_scout\",\"capabilities\":[\"web_scraping\"],\"invite_code\":\"AVOS-OPEN-2026\"}'\n\ncurl -s http://127.0.0.1:8000/agents/<UUID>/heartbeat \\\n  -H 'Authorization: Bearer <JWT>' \\\n  -H 'Content-Type: application/json' \\\n  -d '{\"model\":\"gpt-4\",\"version\":\"0.1.0\",\"status\":\"active\"}'\n\ncurl -s http://127.0.0.1:8000/authorize_action \\\n  -H 'Authorization: Bearer <JWT>' \\\n  -H 'Content-Type: application/json' \\\n  -d '{\"agent_id\":\"<UUID>\",\"action_type\":\"execute_shell_command\",\"action_payload\":{\"command\":\"rm -rf /\"}}'
